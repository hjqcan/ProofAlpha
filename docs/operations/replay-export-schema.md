# Replay Export Schema

## Contract

`autotrade.replay-export.v1` is the offline evidence package for incident and decision replay. It is generated by:

- API: `GET /api/replay-exports`
- CLI: `autotrade export replay-package --json`

Supported filters are `strategyId`, `marketId`, `orderId`, `clientOrderId`, `runSessionId`, `riskEventId`, `correlationId`, `fromUtc`, `toUtc`, and `limit`.

## Top-Level Shape

| Field | Purpose |
| --- | --- |
| `generatedAtUtc` | Export generation timestamp. |
| `contractVersion` | Schema version, currently `autotrade.replay-export.v1`. |
| `query` | Normalized query used for every evidence source. |
| `redaction` | Applied redaction rules and excluded sensitive fields. |
| `completenessNotes` | Explicit gaps such as no matching orders, unavailable readiness, or missing run session. |
| `runSession` | Paper run session metadata when `runSessionId` is supplied and found. |
| `timeline` | Unified audit timeline with sanitized detail JSON. |
| `evidence` | Decisions, order events, orders, trades, positions, and risk events. |
| `strategyConfigVersions` | Config versions observed from run session and decision evidence. |
| `readiness` | First-run readiness snapshot with sanitized evidence. |
| `exportReferences` | API, CLI, and schema references for reproducing the export. |

## Evidence Tables

`evidence.decisions` includes decision id, strategy, action, reason, market, sanitized context JSON, config version, correlation id, execution mode, and run session id.

`evidence.orderEvents` includes event id, order id, client order id, strategy, market, event type, order status, message, sanitized context JSON, correlation id, timestamp, and run session id.

`evidence.orders` includes order id, market, token, strategy, client/exchange ids, correlation id, outcome, side, order type, time-in-force, price, quantity, fill quantity, status, rejection reason, and timestamps.

`evidence.trades` includes trade id, order id, client order id, strategy, market, token, outcome, side, price, quantity, exchange trade id, fee, notional, correlation id, and timestamp.

`evidence.positions` includes position id, market, outcome, quantity, average cost, realized PnL, notional, and timestamp.

`evidence.riskEvents` includes risk event id, code, severity, message, strategy, market, sanitized context JSON, and timestamp.

## Redaction

The replay package intentionally excludes account-sensitive fields that are not needed for offline review:

- `TradingAccountId`
- `OrderSalt`
- `OrderTimestamp`

JSON fields whose names contain `password`, `secret`, `privateKey`, `apiKey`, `authorization`, `credential`, `signature`, `passphrase`, or `mnemonic` are replaced with `[redacted]`. Raw text assignment fragments matching those names are also redacted.

The export keeps operational ids such as order ids, client order ids, strategy ids, market ids, token ids, decision ids, risk event ids, and correlation ids because they are required for replay joins.
