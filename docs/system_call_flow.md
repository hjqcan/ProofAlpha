### 2.2 配置加载顺序

```
优先级（低→高）:
┌──────────────────────────────────────────────────────────────┐
│  1. appsettings.json (基线配置，位于 Autotrade.Cli/)         │
├──────────────────────────────────────────────────────────────┤
│  2. appsettings.{Environment}.json (如 Production/Development)│
├──────────────────────────────────────────────────────────────┤
│  3. --config 指定的文件 (如 appsettings.local.json)          │
├──────────────────────────────────────────────────────────────┤
│  4. 环境变量 (AUTOTRADE_xxx / Polymarket__Clob__ApiKey 等)    │
├──────────────────────────────────────────────────────────────┤
│  5. User Secrets (开发环境)                                   │
└──────────────────────────────────────────────────────────────┘
        ↑
    后者覆盖前者
```

### 2.3 后台服务启动顺序

| 服务                         | 模块       | 作用                                 |
| ---------------------------- | ---------- | ------------------------------------ |
| `TradingRetentionService`    | Trading    | 定期清理过期订单/成交数据            |
| `OrderReconciliationService` | Trading    | 订单对账（Live 模式）/Paper 模式跳过 |
| `KillSwitchService`          | Trading    | 监控 KillSwitch 文件变化             |
| `UnhedgedExposureWatchdog`   | Trading    | 监控未对冲敞口超时                   |
| `DiagnosticsService`         | CLI        | 定期健康检查并告警                   |
| `StrategyRetentionService`   | Strategy   | 定期清理过期决策日志                 |
| `MarketCatalogSyncService`   | MarketData | 定期同步市场目录                     |
| `StrategyManager`            | Strategy   | 策略生命周期管理                     |

```
│                                    ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
```

```
│  │  日志: "一致性校验失败，触发重同步: <assetId>"                           │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                    │                                          │
│                                    ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │  1. 取消订阅该资产                                                       │ │

│  │                                                                          │ │
│  │  2. 清除本地订单簿缓存                                                   │ │
│  │                                                                          │ │
│  │  3. 重新订阅该资产                                                       │ │
│  │     → 日志: "已订阅 1 个资产: <assetId>"                                 │ │
│  │                                                                          │ │
│  │  4. 等待新的 book 快照消息                                               │ │
│  │     → 完整替换本地订单簿                                                  │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                    │                                          │
│                                    ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │  日志: "已请求重同步资产: <assetId>，重同步次数: N"                      │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 DualLegArbitrage 策略决策逻辑

```
┌───────────────────────────────────────────────────────────────────────────────┐
│                         DualLegArbitrageStrategy                               │
│                            双腿套利策略                                        │
│                                                                               │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │                          市场筛选阶段                                     │ │
│  │                                                                          │ │
│  │  从 MarketCatalog 筛选符合条件的市场:                                     │ │
│  │                                                                          │ │
│  │  ✓ 市场活跃 (IsActive = true)                                            │ │
│  │  ✓ 有两个 Token (YES + NO)                                               │ │
│  │  ✓ Liquidity ≥ MinLiquidity (默认 1000)                                  │ │
│  │  ✓ Volume24h ≥ MinVolume24h (默认 500)                                   │ │
│  │  ✓ 有有效订单簿 (BestAsk 存在)                                           │ │
│  │                                                                          │ │
│  │  → 按流动性排序，取前 MaxMarkets (默认 20) 个                             │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                    │                                          │
│                                    ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │                          入场条件判断                                     │ │
│  │                                                                          │ │
│  │  对于每个候选市场:                                                        │ │
│  │                                                                          │ │
│  │  计算: PairCost = YES_BestAsk + NO_BestAsk                               │ │
│  │                                                                          │ │
│  │  如果 PairCost < PairCostThreshold (默认 0.98):                          │ │
│  │    → 套利机会！                                                          │ │
│  │    → 理论上: 买 YES + 买 NO，无论结果如何，到期赎回 $1                    │ │
│  │    → 利润 = $1 - PairCost                                                │ │
│  │                                                                          │ │
│  │  额外检查:                                                                │ │
│  │  ✓ 该市场没有活跃持仓                                                    │ │
│  │  ✓ 已过入场冷却期 (EntryCooldownSeconds)                                 │ │
│  │  ✓ 名义金额 ≤ MaxNotionalPerMarket                                       │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                    │                                          │
│           ┌────────────────────────┴────────────────────────┐                 │
│           │                                                 │                 │
│    有套利机会                                           无套利机会            │
│           │                                                 │                 │
│           ▼                                                 ▼                 │
│  ┌─────────────────────┐                      ┌─────────────────────┐        │
│  │  生成交易意图        │                      │  检查现有持仓        │        │
│  │                     │                      │                     │        │
│  │  TradeIntent:       │                      │  是否需要出场？      │        │
│  │  - Buy YES @ Ask    │                      │  - PairValue > 1.01?│        │
│  │  - Buy NO @ Ask     │                      │  - 持有超时？        │        │
│  │  - Quantity         │                      │  - 单腿敞口超时？    │        │
│  │  - CorrelationId    │                      │                     │        │
│  └─────────────────────┘                      └─────────────────────┘        │
│                                                                               │
└───────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 Paper 模式 vs Live 模式写入差异

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           Paper 模式                                            │
│                                                                                 │
│  ✓ 写入: StrategyRunStates (策略状态)                                           │
│  ✓ 写入: StrategyDecisionLogs (决策日志，如果 DecisionLogEnabled=true)          │
│  ✓ 写入: CommandAuditLogs (CLI 命令审计)                                        │
│  ✓ 写入: OrderEvents (订单事件审计)                                             │
│  ✓ 写入: Trades (成交审计)                                                      │
│  ✗ 不写: Orders (Paper模式订单存内存)                                           │
│  ✗ 不写: Positions (Paper模式不持久化持仓)                                      │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│                           Live 模式                                             │
│                                                                                 │
│  ✓ 写入: 所有表                                                                 │
│  + 订单对账服务会定期同步交易所订单状态到 Orders 表                              │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 6.3 快速验证数据库写入

```sql
-- 1. 检查策略运行状态 (应该有记录，即使没有交易)
SELECT * FROM "StrategyRunStates";

-- 2. 检查 CLI 命令审计 (run 命令要 Ctrl+C 退出后才有)

SELECT * FROM "StrategyDecisionLogs" ORDER BY "TimestampUtc" DESC LIMIT 10;

-- 4. 检查订单事件 (只有下单后才有)
SELECT * FROM "OrderEvents" ORDER BY "TimestampUtc" DESC LIMIT 10;

SELECT * FROM "Trades" ORDER BY "ExecutedAtUtc" DESC LIMIT 10;
```

## 7. 配置热更新

### 7.1 DesiredStates 控制面

```
┌───────────────────────────────────────────────────────────────────────────────┐
│                      跨进程策略控制 (DesiredStates)                            │
│                                                                               │
│  配置文件 (appsettings.local.json):                                            │
│  {                                                                            │
│    "StrategyEngine": {                                                        │
│      "DesiredStates": {                                                       │
│        "dual_leg_arbitrage": "Running",   ← 启动                              │
│        "endgame_sweep": "Stopped"         ← 停止                              │
│      },                                                                       │
│      "ConfigVersion": "v20260106xxxxxx"   ← 版本戳 (变更触发热更新)            │
│    }                                                                          │
│  }                                                                            │
│                                                                               │
│  工作原理:                                                                     │
│  1. StrategyManager 监控 IOptionsMonitor<StrategyEngineOptions>               │
│  2. 检测到 ConfigVersion 变化时，重新评估 DesiredStates                        │
│  3. 如果 DesiredState 从 "Running" 变为 "Stopped" → 停止策略                   │
│  4. 如果 DesiredState 从 "Stopped" 变为 "Running" → 启动策略                   │
│                                                                               │
│  使用场景:                                                                     │
│  - 无需重启进程，动态启停策略                                                  │
│  - 多进程部署时，通过修改配置文件统一控制                                       │
└───────────────────────────────────────────────────────────────────────────────┘
```

### 7.2 KillSwitch 控制面

```
┌───────────────────────────────────────────────────────────────────────────────┐
│                         KillSwitch 紧急停止                                    │
│                                                                               │
│  配置文件 (appsettings.local.json):                                            │
│  {                                                                            │
│    "Risk": {                                                                  │
│      "KillSwitch": {                                                          │
│        "Enabled": true,                                                       │
│        "Token": "abc123"        ← 安全令牌                                    │
│      }                                                                        │
│    }                                                                          │
│  }                                                                            │
│                                                                               │
│  触发方式:                                                                     │
│  1. CLI 命令: dotnet run --project Autotrade.Cli -- killswitch trigger        │
│  2. 修改配置文件中的 Token → KillSwitchService 检测到变化 → 触发 KillSwitch    │
│                                                                               │
│  效果:                                                                         │
│  - 拒绝所有新订单                                                              │
│  - 可选: 取消所有活跃订单                                                      │
│  - 可选: 平掉所有持仓                                                          │
└───────────────────────────────────────────────────────────────────────────────┘
```
