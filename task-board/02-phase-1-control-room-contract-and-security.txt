Phase 1: Control Room Contract and Security
===========================================

Status
------
[DONE]


Intent
------
Make the control room safe to operate locally. Freeze API contracts, command
semantics, local access controls, and command audit behavior before adding more
operator workflows.


Core Boundary
-------------
The Web App can inspect and control the system only through explicit API command
contracts. It must not receive exchange credentials, bypass CLI safeguards, or
mutate trading state without audited command intent.


Accepted Scope
--------------
- control-room API contract tests
- local-only access and command-mode policy
- command confirmation and audit trail
- UI disabled states for read-only, Paper, degraded, and Live modes
- API/Web App build and smoke evidence


Out of Scope
------------
- hosted dashboard
- multi-user account system
- cloud sync
- direct manual order placement
- real exchange credential display


Acceptance Criteria
-------------------
1. Snapshot, markets, market detail, order book, strategy state, and kill switch
   endpoints have contract tests.
2. Control commands can be disabled by configuration and the UI reflects that
   state without pretending an action succeeded.
3. High-risk commands require explicit confirmation and are audit logged.
4. The API documents command mode and data mode in every control snapshot.
5. Browser smoke verifies the control room loads without console errors against
   a local API or deterministic mock.
