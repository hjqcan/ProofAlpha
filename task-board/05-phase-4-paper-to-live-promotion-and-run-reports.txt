Phase 4: Paper-to-Live Promotion and Run Reports
================================================

Status
------
[DONE]


Intent
------
Make Paper mode a measurable proving ground instead of a loose simulation. A
run should produce evidence that explains whether the system is ready for
carefully armed Live operation.


Core Boundary
-------------
Promotion evidence can recommend readiness but must not bypass Live arming,
compliance, account sync, or risk gates.


Accepted Scope
--------------
- Paper run session model
- run report API and exports
- promotion checklist
- PnL, slippage, latency, fill, and exposure attribution
- report UI


Out of Scope
------------
- guarantee of profitability
- automatic Live enablement
- unmanaged real-money experiments
- historical backtest engine


Acceptance Criteria
-------------------
1. Paper runs are grouped into sessions with start, stop, configuration, and
   strategy set.
2. Run reports summarize decisions, orders, fills, PnL, latency, stale data,
   risk events, and unhedged exposure.
3. Promotion checklist has pass/fail reasons and explicit residual risks.
4. Reports export to JSON and CSV without losing audit IDs.
5. UI makes it clear whether evidence came from Paper or Live.
