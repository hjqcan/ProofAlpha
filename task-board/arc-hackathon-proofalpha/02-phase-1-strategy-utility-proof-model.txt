Phase 1: Strategy Utility Proof Model
=====================================

[TODO]


Intent
------
Define and implement the evidence model that lets ProofAlpha prove a strategy
was useful without making false investment claims. This phase prevents a weak
"we put a hash onchain" demo.


Core Boundary
-------------
ProofAlpha proves:

- what the agent knew
- what the agent recommended
- what risk envelope applied
- whether it executed
- what happened afterward

Arc anchors hashes and outcome records. Arc does not decide that a strategy is
profitable.


Accepted Scope
--------------
- Canonical signal document.
- Canonical outcome document.
- Utility metrics.
- Anti-cherry-pick rules.
- Local evidence export.
- Tests for deterministic hashing and redaction.


Out of Scope
------------
- Full statistical backtesting platform.
- Profit guarantee.
- Publishing full reasoning text to public chain.


Proof Documents
---------------
`StrategySignalProofDocument`:

- documentVersion
- agentId
- sourceKind
- sourceId
- strategyId
- marketId
- venue
- createdAtUtc
- configVersion
- evidenceIds
- opportunityHash
- reasoningHash
- riskEnvelopeHash
- expectedEdgeBps
- maxNotionalUsdc
- validUntilUtc
- generatedStrategyPackageHash, optional

`StrategyOutcomeProofDocument`:

- documentVersion
- signalId
- executionId
- runSessionId
- clientOrderIds
- orderEventIds
- tradeIds
- paperOrLive
- status: `Executed`, `Rejected`, `Expired`, `Skipped`, `Failed`
- realizedPnlBps
- slippageBps
- fillRate
- riskRejectReason
- completedAtUtc

[TODO] AH-P1-T003 Define anti-cherry-pick rules.

- Every published signal needs a terminal outcome:
  - executed
  - expired
  - rejected
  - skipped
  - failed
- Outcome deadline must be derived from `validUntilUtc`.
- Reputation includes losing and expired signals.
- Manual deletion of poor signals is not allowed; use tombstone/revoked status.

Acceptance Criteria
-------------------
1. Strategy utility proof can be generated without Arc online.
2. Proof documents are stable and hashable.
3. Proof documents cannot contain obvious secrets.
4. Reputation model includes non-winning outcomes.
5. Demo artifact can explain "why this strategy looked useful."
