Phase 2: Arc Contract Suite Foundation

------

config validation tests, solution build, and evidence note.


Intent
------
Build the contract and deployment foundation for the paid-agent loop.

--------------

- Contract tests.
- Deployment scripts.
- Deployment artifact format.
- Backend config schema.
- ArcSettlement bounded context skeleton.


Contract Suite
--------------
Minimum contracts:

- `SignalRegistry`
- `StrategyAccess`
- `PerformanceLedger`
- `RevenueSettlement`

Optional/stretch:

- `ExecutionPermission`
- `AgentReputation`

[DONE] AH-P2-T003 Implement `SignalRegistry`.

Required:

- duplicate `signalId` rejected
- zero agent rejected
- empty venue rejected
- signal metadata stored or event emitted with enough fields
- read method for public signal summary

[DONE] AH-P2-T004 Implement `StrategyAccess`.

Required:

- accepts configured ERC20/USDC token address
- plan id maps to price/duration/strategy key
- `subscribe(strategyKey, planId)`
- `hasAccess(user, strategyKey)`
- `accessExpiresAt(user, strategyKey)`
- emits `StrategySubscribed`

[DONE] AH-P2-T005 Implement `PerformanceLedger`.

Required:

- records terminal outcome for signal id
- prevents multiple terminal outcomes unless explicit correction event exists
- emits `OutcomeRecorded`
- includes status, realized pnl bps, slippage bps, and outcome hash

[DONE] AH-P2-T006 Implement `RevenueSettlement`.

MVP:

- event-only settlement journal is acceptable
- records settlement id, signal id, gross amount, shares, token, and timestamp
- no actual fund distribution required for MVP
