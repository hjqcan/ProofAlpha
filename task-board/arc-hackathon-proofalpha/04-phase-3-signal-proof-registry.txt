Phase 3: Signal Proof Registry
==============================

Status
------
[TODO]


Intent
------
Publish agent strategy signals to Arc so users can verify when a signal was
created and what evidence/reasoning hash backed it.


Accepted Scope
--------------
- Signal publication application service.
- Local publication intent/result records.
- Arc EVM publisher.
- CLI publish/list commands.
- API read model.
- Tests for idempotency, disabled mode, and duplicate signal handling.


Out of Scope
------------
- Automatic publication from every strategy cycle.
- Chain writes from `StrategyRunner`.
- Publishing full reasoning content onchain.

Acceptance Criteria
-------------------
1. A signal proof can be published to Arc testnet or local EVM.
2. Re-running publication does not create duplicate local records.
3. API and CLI expose publication state.
4. Signal proof links to canonical proof document hash.
5. Failure modes are visible and auditable.
