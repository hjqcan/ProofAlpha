Phase 3: Signal Proof Registry
==============================

Status
------
[DONE]

Current verified state:

- canonical signal proof publication path is implemented
- source-driven proof publication is implemented for approved opportunities and
  strategy decisions
- local publication records are persisted and idempotent
- disabled Arc mode records explicit skipped state and does not write chain
- Hardhat-backed EVM publisher path is implemented
- local in-process Hardhat demo publishes `SignalPublished`
- CLI and API expose publication records and source-driven publish commands


Intent
------
Publish agent strategy signals to Arc so users can verify when a signal was
created and what evidence/reasoning hash backed it.


Accepted Scope
--------------
- Signal publication application service.
- Local publication intent/result records.
- Arc EVM publisher.
- CLI publish/list commands.
- API read model.
- Tests for idempotency, disabled mode, and duplicate signal handling.


Out of Scope
------------
- Automatic publication from every strategy cycle.
- Chain writes from `StrategyRunner`.
- Publishing full reasoning content onchain.

[DONE] AH-P3-T004 Connect OpportunityDiscovery source.

- Include evidence ids and compiled policy hash.
- Include valid-until check.
- CLI and API can resolve `--source opportunity --id <guid>` without requiring
  `--proof-file`.

[DONE] AH-P3-T005 Connect StrategyDecision source.

- Allow publishing a selected strategy decision only if it has:
  - market id
  - strategy id
  - decision context
  - risk envelope
  - non-empty reasoning/context hash
- Generated proof derives reasoning/context hash and risk envelope hash from
  the decision context.
- CLI and API can resolve `--source decision --id <guid>` without requiring
  `--proof-file`.

[DONE] AH-P3-T006 Add CLI commands.

Commands:

```powershell
dotnet run --project .\interfaces\Autotrade.Cli\Autotrade.Cli.csproj -- arc signal publish --source opportunity --id <guid> --actor <actor> --reason <reason>
dotnet run --project .\interfaces\Autotrade.Cli\Autotrade.Cli.csproj -- arc signal publish --source decision --id <guid> --actor <actor> --reason <reason>
dotnet run --project .\interfaces\Autotrade.Cli\Autotrade.Cli.csproj -- arc signal list --limit 20
dotnet run --project .\interfaces\Autotrade.Cli\Autotrade.Cli.csproj -- arc signal show --signal-id <signal-id>
```

Canonical proof-file publication remains available as an explicit fallback:

```powershell
dotnet run --project .\interfaces\Autotrade.Cli\Autotrade.Cli.csproj -- arc signal publish --proof-file <path> --source opportunity --id <id> --actor <actor> --reason <reason>
```

Acceptance Criteria
-------------------
1. A signal proof can be published to Arc testnet or local EVM.
2. Re-running publication does not create duplicate local records.
3. API and CLI expose publication state.
4. Signal proof links to canonical proof document hash.
5. Failure modes are visible and auditable.
