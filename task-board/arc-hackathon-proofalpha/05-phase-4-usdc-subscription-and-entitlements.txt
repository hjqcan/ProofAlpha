Phase 4: USDC Subscription and Entitlements
===========================================

Status
------
[DONE]

Current verified state:

- StrategyAccess deployment config exists under `ArcSettlement:Contracts`.
- local Arc deployment artifact includes `paymentToken` for local TestUsdc.
- backend subscription plan config is bound and validated.
- entitlement mirror/access contracts are implemented in ArcSettlement.
- JSON-file entitlement mirror store is registered for local/demo operation.
- access reader returns active, expired, missing-wallet, invalid-wallet,
  missing-strategy, not-found, and disabled states.
- manual known-transaction sync can ingest a `StrategySubscribed` tx hash into
  the mirror with wallet/strategy/plan/expiry/block audit fields.
- CLI exposes `arc access plans`, `arc access status`, and `arc access sync`.
- API exposes plans, access status, and sync endpoints.
- local EVM demo produces `StrategySubscribed` evidence.
- entitlement status artifact shows active access and `canViewSignals: true`.


Intent
------
Make USDC payment on Arc unlock a real ProofAlpha capability. This is the
commercial core of the hackathon project.


Accepted Scope
--------------
- StrategyAccess contract integration.
- Subscription plan model.
- Entitlement mirror in backend.
- Manual refresh/sync command.
- API read model.
- Tests for entitlement gating.


Out of Scope
------------
- Production billing.
- Custody.
- Mainnet token handling.
- Real tax/accounting.

- `PublishSignal`
- `RecordSettlement`

5. Entitlement logic is not only visual; at least one backend path uses it.
