Phase 7: Performance Ledger and Agent Reputation
================================================

Status
------
[DONE]


Intent
------
Use blockchain records to prove strategy usefulness over time. The agent should
build reputation from all signal outcomes, not from selective screenshots.


Accepted Scope
--------------
- Outcome recording service.
- PerformanceLedger contract integration.
- Local reputation read model.
- Metrics in API and WebApp.
- Tests for expired/loss/rejected inclusion.


Out of Scope
------------
- Full quantitative research platform.
- Guaranteed prediction of future performance.
- Editing or deleting bad outcomes.


Outcome Statuses
----------------
- `ExecutedWin`
- `ExecutedLoss`
- `ExecutedFlat`
- `RejectedRisk`
- `RejectedCompliance`
- `Expired`
- `SkippedNoAccess`
- `FailedExecution`
- `CancelledOperator`

Acceptance Criteria
-------------------
1. One signal can be recorded with a terminal outcome on Arc.
2. Agent reputation changes after outcome record.
3. Loss/expired/rejected outcomes are represented.
4. WebApp can show "proof of usefulness" without guaranteeing future profit.
5. Demo includes both signal tx and outcome tx.
