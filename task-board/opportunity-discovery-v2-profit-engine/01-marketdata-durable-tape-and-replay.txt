Goal
----
Persist enough append-only market data to backtest and replay opportunity
decisions without forward-looking data.


Why This Comes First
--------------------
Opportunity quality cannot be measured from current in-memory order book state.
The system must record historical prices, top-of-book, depth snapshots, trades,
and resolution events before any scoring or Live promotion claim is meaningful.


Scope
-----
- Add append-only MarketData entities:
  - `MarketPriceTick`
  - `OrderBookTopTick`
  - `OrderBookDepthSnapshot`
  - `ClobTradeTick`
  - `MarketResolutionEvent`
- Add contract DTOs and repository interfaces:
  - `IMarketTapeWriter`
  - `IMarketTapeReader`
  - `IMarketReplayReader`
- Add EF Core mappings and indexes in `MarketDataContext`.
- Add idempotent upsert/append semantics for duplicate WebSocket and polling
  events.
- Add deterministic replay primitives for top-of-book slices.
- Add a minimal backtest runner that computes executable entry/exit opportunity
  performance from persisted top-of-book ticks.


Out of Scope
------------
- Source registry.
- LLM prompt changes.
- Live publishing.
- Full order book depth fill model. This task stores depth snapshots and exposes
  them, but promotion-grade fill modeling belongs to ODV2-T005.


Implementation Steps
--------------------
1. Domain and contract model
   - Create MarketData tape entity classes under
     `Autotrade.MarketData.Domain/Entities`.
   - Create contract records under
     `Autotrade.MarketData.Application.Contract/Tape`.
   - Keep DTOs immutable and avoid exposing EF entities across the contract
     boundary.

2. EF persistence
   - Add `DbSet` properties.
   - Configure numeric precision for price, size, fee, notional, and quantity.
   - Configure `RawJson` as `jsonb` for Postgres.
   - Add indexes:
     - `MarketPriceTicks(MarketId, TokenId, TimestampUtc)`
     - `OrderBookTopTicks(MarketId, TokenId, TimestampUtc)`
     - `OrderBookDepthSnapshots(MarketId, TokenId, TimestampUtc)`
     - `ClobTradeTicks(MarketId, TokenId, TimestampUtc)`
     - `MarketResolutionEvents(MarketId, ResolvedAtUtc)`
   - Add unique dedup indexes where source event identity exists:
     - top ticks: `(MarketId, TokenId, SourceSequence)`
     - depth snapshots: `(MarketId, TokenId, SnapshotHash, TimestampUtc)`
     - trades: `(MarketId, TokenId, ExchangeTradeId)`

3. Repository and replay APIs
   - Implement `EfMarketTapeRepository`.
   - `Append*Async` must ignore exact duplicates and never mutate existing tape.
   - `GetTopTicksAsync` must sort ascending by timestamp.
   - `GetReplaySliceAsync` must clamp data to `asOfUtc`.

4. Ingestion wiring
   - Add a lightweight `MarketTapeRecorder` that converts CLOB WebSocket events
     into tape DTOs.
   - Wire it from `OrderBookSynchronizer` only after repository tests are green.
   - On write failure, log and emit metrics, but do not break the in-memory
     order book path.

5. Backtest primitive
   - Add `MarketReplayBacktestRunner` with top-of-book degraded mode.
   - Input:
     - market id
     - token id
     - side/outcome
     - entry max price
     - take-profit price
     - stop-loss price
     - quantity
     - max notional
     - from/to/as-of timestamps
   - Output:
     - entry fill timestamp and price
     - exit fill timestamp and price
     - gross PnL
     - estimated fees
     - net PnL
     - replay seed
     - completeness notes
