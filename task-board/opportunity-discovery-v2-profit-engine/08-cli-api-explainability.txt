ODV2-T008: CLI/API Explainability and Operator Controls
=======================================================

Status
------
[DONE]


Goal
----
Expose score, replay, promotion, Live status, suspension, and explanation flows
without requiring direct database inspection.


CLI Commands
------------
- `opportunity score --id <id>`
- `opportunity replay --id <id>`
- `opportunity promote --id <id>`
- `opportunity live-status`
- `opportunity suspend --id <id> --reason <reason>`
- `opportunity explain --id <id>`


Explain Output
--------------
- evidence snapshot
- source profile versions
- source conflicts
- market tape timestamps
- price and order book snapshot
- model and prompt version
- score version
- score breakdown
- expected EV
- executable capacity
- gate results
- allocation state
- risk and compliance reasons


Acceptance
----------
- CLI and API return the same contract shape.
- Explain output is redacted for secrets.
- Operators can suspend an opportunity without deleting evidence.


Implementation Notes
--------------------
- Added `IOpportunityOperatorService` shared contracts for score, replay,
  promote, Live status, suspend, and explain responses.
- Added `OpportunityOperatorService` to assemble point-in-time evidence,
  source profile versions, conflicts, market tape ids, prompt/model/score
  versions, score breakdown, expected EV, executable capacity, gates,
  evaluation runs, executable policy, allocation state, and risk/compliance
  reasons.
- Explain output redacts sensitive JSON/text keys such as API keys, private
  keys, secrets, passwords, authorization values, bearer values, mnemonics, and
  seed phrases.
- Added `api/opportunities` endpoints for score, replay, promote, live-status,
  suspend, and explain; responses use the same application contracts used by
  the CLI.
- Added CLI commands:
  - `opportunity score --id <id>`
  - `opportunity replay --id <id>`
  - `opportunity promote --id <id>`
  - `opportunity live-status`
  - `opportunity suspend --id <id> --reason <reason>`
  - `opportunity explain --id <id>`
- Manual suspend uses the existing Live allocation suspension service, so it
  suspends hypothesis, policy, allocation, and cancels matching open orders
  where order/execution services are available, while leaving evidence rows
  untouched.
