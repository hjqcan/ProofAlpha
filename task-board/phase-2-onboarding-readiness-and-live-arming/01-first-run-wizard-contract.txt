Goal
----
Define the operator journey from fresh checkout to a safe Paper run.

Acceptance
----------
- Wizard contract lists required checks, optional checks, and blocked checks.
- Checks cover database, migrations, API reachability, market data, account
  sync, compliance, execution mode, and risk limits.
- Response is machine-readable for UI and CLI rendering.
- Missing configuration produces next-action guidance, not a generic failure.
