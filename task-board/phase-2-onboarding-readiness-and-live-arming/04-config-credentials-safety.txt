Phase 2.4: Config and Credential Safety
=======================================

Status
------
[DONE]

Goal
----
Make configuration and credential problems discoverable without exposing
secrets.

Acceptance
----------
- Diagnostics can say whether required credential fields are present without
  returning their values.
- Logs and API responses redact private keys, API secrets, and bearer material.
- Unsafe Live parameter combinations remain visible and blocked unless policy
  explicitly permits them.
- Local config examples remain Paper-first.
- Tests cover redaction and missing-secret paths.

Completion Notes
----------------
- Added shared credential-presence diagnostics that report only `present` or
  `missing` for required exchange credentials.
- Added shared secret redaction for JSON, key-value, Authorization, and bearer
  material.
- Readiness probes now expose missing credential field names without returning
  configured values.
- API exception responses and logs redact known secret-bearing strings before
  they can leave the process boundary.
- Default CLI and API appsettings remain Paper-first.
